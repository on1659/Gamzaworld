# 🎮 Gamzaworld

귀여운 캐릭터 기반 타이밍/피지컬 미니게임 천국

## 🎯 컨셉

- **타겟층**: 전연령, 타이밍 게임 & 피지컬 게임 선호
- **게임 방식**: 싱글 게임 + 글로벌 랭킹 경쟁
- **핵심 차별점**: 업적/아이콘/레벨 시스템으로 성장 과시
- **게임 철학**: 간단하지만 중독성 있는 타이밍/반사신경 게임. 속도감 필수!

## 🕹 게임 목록 (유동적)

**현재:**
1. **타이밍 점프** ✅ - 장애물 피하며 최대한 멀리

**후보 (추가/변경/제거 자유):**
2. **리듬 탭** - 떨어지는 노트 타이밍 맞추기
3. **스택 타워** - 타이밍 맞춰 블록 쌓기
4. **빠른 반응** - 색깔/방향 순간 판단
5. **회피 러시** - 끊임없이 쏟아지는 장애물 피하기
6. **타이밍 슬라이스** - 정확한 타이밍에 자르기

💡 **개발 시 고려:**
- 컨셉에 맞는가? (타이밍/피지컬/반사신경)
- 속도감 있는가?
- 중독성 있는가?
- 다른 게임과 차별화되는가?
→ 맞으면 추가, 아니면 과감히 제거!

## 🏗 아키텍처 (멀티플레이어 고려)

### 게임 로직 분리 (서버/클라이언트 공용)
```
/src/games/
  /shared/
    - GameSession.js        ← 베이스 세션 클래스
  /timing-jump/
    - logic.js              ← 게임 로직 (서버/클라이언트 공용)
    - TimingJump.svelte     ← UI 렌더링만
```

### 서버 게임 세션 관리
```
/server/game/
  - GameSessionManager.js   ← 세션 생성/관리/업데이트
```

### 게임 플로우

**싱글 모드:**
1. 클라이언트 → `game:start` → 서버
2. 서버: 세션 생성 + 게임 루프 시작
3. 클라이언트: 입력 전송 (`game:input`)
4. 서버: 게임 로직 실행 → 상태 브로드캐스트 (`game:state`)
5. 클라이언트: 상태 받아서 렌더링만

**1:1 대결 모드 (추후):**
- 같은 구조, 두 플레이어 동시 처리
- 서로의 상태를 `game:opponent-state`로 실시간 공유

### 점수 검증
- ✅ 서버에서 점수 계산 (클라이언트는 렌더링만)
- ✅ 치팅 방지
- ✅ 멀티 대응 가능

## 🎨 리소스 교체 방법

리소스(캐릭터, UI, 사운드)는 모듈화되어 쉽게 교체 가능:

1. `/public/assets/characters/custom` 폴더 생성
2. 이미지 파일 추가 (idle.png, jump.png, celebrate.png, sad.png)
3. `/src/config/theme.js`에서 `current: 'custom'`으로 변경

## 📦 설치 및 실행

```bash
# 설치
npm install

# 개발 서버 (프론트엔드)
npm run dev

# 백엔드 서버 (개발)
npm run server:dev

# 프로덕션 빌드
npm run build

# 프로덕션 실행
npm start
```
